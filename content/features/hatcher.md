# 孵化器（Hatcher）— 自己建 Bot

> **孵化器是把「通用 AI」變成「你的專屬數位員工」的地方。**
>
> 原生波特人什麼都能做一點，但孵化器讓你針對特定業務情境，打造一個專精某件事的 AI Bot——它知道你的術語、遵守你的規則、只用你指定的工具。

## 孵化器是什麼？

孵化器（Hatcher）是波特人的 Bot 建立器。它像一個「AI 實驗室」：你定義角色、知識、工具，孵化器幫你組裝出一個專屬的 AI 助理。

**不需要寫程式。** 整個過程就是填表單、上傳檔案、勾選工具。

### 原生波特人 vs 孵化出的 Bot

| | 原生波特人 | 孵化出的 Bot |
|--|----------|------------|
| **誰能用** | 所有人，進入就能用 | 你建的，可分享給指定的人 |
| **知識範圍** | 通用知識 + 網路搜尋 | 通用知識 + **你上傳的專屬文件** |
| **行為規則** | 平台預設 | **你定義的角色、語氣、限制** |
| **工具能力** | 平台預設工具 | **你選擇的工具組合** |
| **適合場景** | 臨時問答、快速分析 | **反覆使用的專業流程** |

## 你可以設定什麼

孵化器的設定分為五個區塊：

### 一、基本設定

| 設定 | 說明 | UI 標籤 |
|------|------|--------|
| **Bot 名稱** | 給你的 Bot 一個好記的名字 | 「名稱」 |

### 二、提示工程（最核心的部分）

提示詞模板是 Bot 的「大腦指令」，定義它是誰、怎麼說話、能做什麼不能做什麼。

你可以在提示詞中設定：

| 設定面向 | 說明 | 範例 |
|---------|------|------|
| **角色定義** | Bot 的身份和專業 | 「你是內政部的 SQL 專家，專門處理人口統計資料庫查詢」 |
| **互動規則** | 對話風格和氛圍 | 「使用台灣繁體中文，語氣溫暖但專業」 |
| **限制條件** | 不能做的事 | 「禁止使用簡體字」「不做法律解釋，只引導到主責機關」 |
| **任務目標** | Bot 存在的目的 | 「幫使用者把自然語言描述轉成可執行的 SQL 查詢」 |
| **輸出格式** | 回答的結構 | 「每次回答都附上引用的法條編號」 |

### 三、Bot 機制

| 模式 | 說明 | 適合場景 |
|------|------|---------|
| **基礎模式** | 純對話，直接用提示詞回應 | 單純問答、文字處理 |
| **多代理模式（Agent）** | 可呼叫工具，自主判斷何時用哪個工具 | 需要上網、分析檔案、產出圖表的複合任務 |

選擇多代理模式後，Bot 獲得「自主思考」能力——它會判斷你的問題需要哪些工具，自動組合執行。

**模型選擇**：基礎模式和多代理模式有各自獨立的模型池，你可以為不同需求選擇最適合的模型。

### 四、MCP 工具箱（僅多代理模式）

勾選你要給 Bot 的工具。常見工具包括：

| 工具 | 功能 | 使用場景 |
|------|------|---------|
| `web_search` | 即時聯網搜尋 | 查最新資訊、新聞 |
| `scrape` | 網頁內容擷取 | 讀取指定網頁 |
| `chat_with_pdf` | 深度 PDF 分析 | 合約、論文、報告問答 |
| `chat_with_imgs` | 多圖辨識分析 | 圖片內容辨識 |
| `generate_image` | AI 圖片生成 | 視覺素材產出 |
| `create_plotly_chart` | 互動式圖表 | 數據視覺化 |
| `create_mermaid_diagram` | 流程與架構圖 | 系統設計、專案管理 |
| `create_html_page` | 客製化網頁 | 報告頁面、儀表板 |

多個工具可以組合使用。例如：Bot 先上網搜尋資料，再整理成圖表，最後產出一個網頁報告。

### 五、知識庫

上傳你的專屬文件，Bot 會根據這些資料回答問題。

| 功能 | 說明 |
|------|------|
| **新增檔案** | 上傳多個檔案（上限 200MB） |
| **支援格式** | PDF、Word、CSV、Excel 等 |
| **檔案管理** | 查看已上傳檔案、大小、刪除 |
| **語意搜尋** | Bot 用向量搜尋找到最相關的內容 |

## 建立步驟（5 分鐘完成）

```
步驟 1：點選頂部導航列的「孵化器」分頁
  ↓
步驟 2：按「建立 BOT」
  ↓
步驟 3：填寫名稱（例如「內政部簽證審查」）
  ↓
步驟 4：撰寫提示工程（定義角色、規則、任務）
  ↓
步驟 5：選擇 BOT 機制
  ├── 基礎：純對話
  └── 多代理：可用工具 → 勾選 MCP 工具
  ↓
步驟 6：上傳知識庫檔案（可選）
  ↓
步驟 7：在右側測試區即時測試
  ↓
完成！系統每 3 秒自動儲存
```

**儲存狀態指示**：畫面會顯示「尚未儲存」→「儲存中」→「於 XX:XX 儲存成功！」

## 分享你的 Bot

每個 Bot 都有一個**專屬網址**（`botrun.ai/b/你的BotID`）。

| 功能 | 說明 |
|------|------|
| **複製專屬網址** | 一鍵複製，貼給同事就能用 |
| **指定分享** | 分享給特定使用者（需權限驗證） |
| **取消分享** | 隨時收回權限 |
| **My Playbook** | 瀏覽他人分享的 Bot，複製來用 |

對方收到連結後，會看到 Bot 的介紹頁面，點「新增並開啟對話」即可開始使用。

## 真實案例：客戶用孵化器做了什麼

### 案例一：內政部「波夫」— 簽證審查 30 秒完成

**情境：** 宗禮司每年審查大量外籍人士來台研修案，每案人工審查需要 3 小時。

**孵化器設定：**
- **角色**：簽證審查專員
- **知識庫**：八類公文草稿範本、審核表格範本
- **工具**：PDF 分析（辨識護照）、自動文件合併（「迴文針」功能）

**Bot 能做的事：**
1. 上傳申請文件（最多 10 人團體）
2. 自動辨識護照中的姓名、護照號碼、出生年月日
3. 自動比對台灣參與比例（需達 50%）
4. 產出完整公文（3 頁）+ 審核表格（1 頁）
5. 自動生成「長官核稿清單」含頁碼對照

**成效：** 人工 3 小時 → AI **30 秒至 1 分鐘**

---

### 案例二：內政部「波庫」— SQL 專家

**情境：** 統計處需要分析少子化根因、住宅與婚齡關聯性，但 SQL 開發耗時。

**孵化器設定：**
- **角色**：SQL 專家
- **知識庫**：資料庫 schema 文件
- **提示詞**：定義資料庫結構、欄位命名規則

**成效：** SQL 開發從**兩天→三小時**

---

### 案例三：調查局「黃喉貂」— 宣導圖製作

**情境：** 調查員需要製作宣導圖，但沒有設計人力。

**孵化器設定：**
- **角色**：宣導圖設計師
- **工具**：AI 圖片生成
- **提示詞**：定義調查局品牌風格、吉祥物（黃喉貂）使用規範

**成效：** 調查員自己就能產出專業宣導圖

---

### 案例四：市集案「AIEC 會議紀錄」— 自動整理

**情境：** 大量會議紀錄需要整理成統一格式。

**孵化器設定：**
- **角色**：會議紀錄整理專員
- **工具**：網頁製作（產出 HTML 報告）
- **提示詞**：定義會議紀錄格式、重點摘要規則

**成效：** 產出可直接分享的 HTML 格式會議紀錄

## 進階使用

### 搭配高品味題庫

孵化器 + [高品味題庫](../getting-started/index.html#第二站高品味題庫--導入成功的關鍵) 是最佳組合：

1. 在孵化器中建立 Bot，定義角色和工具
2. 用高品味題庫提供範例（六欄位：動機、效益、提問、答案、原始資料、鑑別原則）
3. Bot 自動從範例中學會怎麼回答
4. 不需要手工打磨提示詞，題庫做好了提示工程就完美了

### 知識庫（RAG）

> **上傳你的文件，AI 根據你的資料回答。**

- **上傳多種格式**：PDF、Word、CSV、Excel
- **上傳多個檔案**：建立完整的知識庫
- **中文問答**：用自然語言問問題，AI 從你的文件中找答案
- **搜尋增強**：可開啟搜尋功能，讓 Bot 同時參考網路資料和你的文件
- **網域過濾**：限制搜尋結果只來自指定網站
- **引證來源**：回答時標註出處，可驗證

### 知識圖譜（botrun-graph）

RAG 的進化版，建立文件之間的語意關係圖：

| 項目 | GraphRAG | botrun-graph |
|------|----------|-------------|
| API 請求量 | 449 次 | 約 22 次（1/20） |
| 成本 | USD 11.62 | USD 0.62 |
| 成本比 | — | **便宜 19 倍** |

適合企業級大量文件知識庫，以低成本實現深層語意檢索。曾應用於百萬公文檢索專案。

### MCP 工具整合

> **讓你的 Bot 不只會聊天，還會「動手做事」。**

MCP（Model Context Protocol）讓 Bot 能呼叫外部工具。當你把 Bot 設為 Agent 模式，它可以：

- **上網搜尋**：查即時資訊
- **存取 Google Drive**：讀取和寫入你的雲端文件
- **呼叫自訂 API**：串接你自己的系統
- **執行資料分析**：處理 CSV、產出圖表
- **多工具組合**：一個指令，Bot 自動組合多個工具完成任務

設定方式：
1. 在孵化器中將 Bot 設為「Agent 模式」
2. 系統自動載入可用的 MCP 工具列表（`/v2/hatch/mcp-tools`）
3. 勾選你需要的工具，多個工具配置自動合併為 JSON
4. Bot 會在對話中自動判斷何時該用哪個工具

### 多代理模式（Multi-Agent）

Bot 有兩種運作模式：

**基礎模式（Basic）**：標準 LLM 對話，適合一般問答與文件分析。

**Agent 模式（Multi-Agent）**：啟用後，Bot 可自主使用多種工具來回答問題：
- 上網搜尋即時資訊
- 解析上傳的文件
- 繪製圖表（Banana 圖表生成）
- 執行程式碼
- 呼叫 MCP 工具
- 產出 PDF 報告

### 資料即代理（Data as Agents）

波小隊的多代理架構將資料轉化為具備行動能力的代理：
- **記憶能力** — 跨對話保留上下文
- **上網搜尋** — 取得即時資訊
- **程式碼執行** — 在隔離環境中執行運算
- **PDF 報告產出** — 自動生成結構化報告

### 自動化流程

設定 Bot 在接收到特定指令時，自動執行一系列動作：
- 搜尋資料 → 整理成表格 → 儲存成檔案
- 讀取 PDF → 擷取關鍵數據 → 產出圖表 → 生成報告網頁

## 孵化器常見問題

| 問題 | 解答 |
|------|------|
| 不會寫提示詞怎麼辦？ | 用[高品味題庫](../getting-started/index.html#第二站高品味題庫--導入成功的關鍵)提供範例，AI 自動學會 |
| 建立的 Bot 會被別人看到嗎？ | 不會，除非你主動分享 |
| 可以建幾個 Bot？ | 沒有數量限制 |
| 上傳的檔案有大小限制嗎？ | 單檔上限 200MB |
| 編輯到一半會遺失嗎？ | 不會，系統每 3 秒自動儲存 |
| 模型可以換嗎？ | 可以，基礎模式和多代理模式各有獨立模型池 |
